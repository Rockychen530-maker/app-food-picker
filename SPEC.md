# 「就決定是你了」美食推薦 App - 專案規格書

## 一、專案概述

**名稱**：就決定是你了  
**定位**：AI 美食推薦 + 街景隱藏店家挖掘  
**核心價值**：解決「三餐選擇困難症」，一秒決定吃什麼  
**MVP 形式**：手機版 Web App（響應式設計）

---

## 二、核心功能規格

### 2.1 個人喜好篩選器

| 篩選項目 | 設計 |
|---------|------|
| **預算範圍** | 滑桿：<$100 / $100-300 / >$300 |
| **料理類型** | 複選標籤：中式、日式、韓式、美式、義式、在地小吃、異國料理 |
| **距離範圍** | 動態半徑：500m、1km、5km |
| **特殊需求** | 適合家庭（大桌/兒童座椅）、可外帶、有停車位 |

### 2.2 一鍵決定按鈕

- **視覺**：精靈球（Pokeball）造型按鈕，置於畫面中央
- **行為**：按下後系統根據篩選條件 + 廣告加權演算法，直接推薦一家餐廳
- **動畫**：點擊後旋轉動畫 → 彈出推薦卡片

### 2.3 雙層地圖

| 圖層 | 來源 | 標示 |
|------|------|------|
| 圖層 A | Google Maps 標準店家 | 藍色圖示 |
| 圖層 B | AI 街景偵測隱藏店家 | 金黃色神秘圖示（獨家） |

### 2.4 推薦卡片

- 店家名稱、評分、距離
- 料理類型、價格區間
- Google Maps 連結
- 「再轉一次」按鈕

---

## 三、技術架構

### 3.1 前端框架

- **Framework**：Next.js 16.2.10（App Router）
- **Styling**：Tailwind CSS
- **地圖**：Google Maps JavaScript API
- **動畫**：Framer Motion
- **部署**：Vercel（免費方案）

### 3.2 後端 / API

- **API Route**：Next.js API Routes（無需獨立伺服器）
- **資料庫**：SQLite（本地檔案，MVP階段）
- **地圖數據**：Google Places API

### 3.3 AI 街景偵測（Phase 2）

- **OCR**：Tesseract OCR（輕量化招牌文字辨識）
- **LLM 推理**：Gemini API（僅在 OCR 觸發後使用，降低 Token 費用）
- **流程**：OCR掃描街景 → 發現疑似餐廳 → 比對Google Places → 無資料才送LLM

### 3.4 廣告系統

- **初期**：Google AdMob 原生廣告
- **中後期**：自營商家自助投放系統
